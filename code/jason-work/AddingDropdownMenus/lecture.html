CSS Backgrounds

CSS supports several useful background related properties.

  • background-color: assign the background color
  • background-image: url() applies an image as the background
  • background-repeat: repeat | repeat-x | repeat-y | no-repeat determines how a background image will repeat display
  • background-position: <percentage> | <length> | left | center | right the position can be assigned with either pixels or percent
    background-position: 100px 100px; using pixels, you can position the x y starting point for the background image. 0px 0px is the default
    background-position: 50% 0px; using percentages you can center a background image, or with varying percentages can create a parallax background effect

The pattern to follow when using the background property to define several space delimited values at once:

background: <background-color> | <background-image> | <background-repeat> | <background-attachment> | <background-position>

Sprites

Each image on your website requires a separate request/response between the client and server. Using one larger image instead of several smaller images can result in a more effiecient client/server transaction, since it reduces the number of requests/responses that need to occur.

This is especially useful for images that are NOT an important part of your foreground content, but sprites perfect for icons, textures and images used purely for aesthetics

  1. use a sprite generator or an image program like photoshop to create a strip or grid of images. this is also an ideal time to determine the correct offsets
  2. use an HTML tag ('div' or 'p' or etc) any block element
  3. define a fixed height and width (large enough to show the image you need to display)
  4. use the url and position offset values for example:
  • background: url("myimage.jpg") xOffset yOffset;
  • background-image: url()
  • background: url("myimage.jpg") 0px 0px; places myimage.jpg in the top left corner ( 0px 0px )
  • background: url("myimage.jpg") 50px 0px; places myimage.jpg at the top, but shifts the start of the image 50 pixels to the right (offset 50px)
  • background: url("myimage.jpg") -50px 0px; places myimage.jpg at the top, but shifts the start of the image 50 pixels to the left (offset -50px)

Advantages of using sprites

  • fewer http requests between server and client
  • a single composite image is almost always smaller in filesize compared to the sum size of the images made separately
  • perfect for icons, buttons, and graphics that contribute to the aesthetic of the page

Disadvantages of using sprites

  • not all images should be displayed as backgrounds! important content graphics should always use the img tag. this way the images will contribute to search engine optimization and maximize accessability
  • time spent building the composite sprite image(s) increases
  • time coding and maintaining the CSS increases
